\subsection{Detailed Description}
Stores the database of poses for Pose Estimation. 

This class is used internally by \hyperlink{classPoseEstimation}{Pose\-Estimation}, however it can be used to create or load multiple databases and test pose estimation with them. I.\-e. with set\-Database() method of \hyperlink{classPoseEstimation}{Pose\-Estimation}. Some examples\-: 
\begin{DoxyCode}
\hyperlink{classPoseDB}{PoseDB} database; \textcolor{comment}{//empty database}
database.\hyperlink{classPoseDB_aa4209b95406472e4eea23cd9bc12eb66}{load}(\textcolor{stringliteral}{"LOCATION"}); \textcolor{comment}{//load a database from a location}
\hyperlink{classPoseDB}{PoseDB} database2(\textcolor{stringliteral}{"LOCATION2"}); \textcolor{comment}{//create another database and load it from another location}
database = database2; \textcolor{comment}{//Now database holds a copy of database2}
database2.\hyperlink{classPoseDB_a608ac44000f275c1085c208e7906b95d}{clear}(); \textcolor{comment}{//database2 is now empty}
database2.create(\textcolor{stringliteral}{"PCD\_FILES\_LOCATION"}); \textcolor{comment}{//create a new database from scratch using the poses found at
       supplied location}
\textcolor{comment}{//...}
\hyperlink{classPoseEstimation}{PoseEstimation} prova; \textcolor{comment}{//create a pose estimation with default parameters}
prova.\hyperlink{classPoseEstimation_a0ace84f98f2bab31e64ffaa4eefdf402}{setDatabase}(database); \textcolor{comment}{//tell prova to use the first database}
\textcolor{comment}{//...  estimate}
prova.\hyperlink{classPoseEstimation_a0ace84f98f2bab31e64ffaa4eefdf402}{setDatabase}(database2); \textcolor{comment}{//now use the second database}
\textcolor{comment}{//... estimate again with another database}
\end{DoxyCode}

This method uses the provided set of parameters to create the database, erasing any previously loaded databases. Please note that\-:
\begin{DoxyItemize}
\item Constructing a database from scratch can take several minutes at least.
\item In order to use this method, P\-C\-D files must follow a naming convention, that is obj\-Name\-\_\-latitude\-\_\-longitude.\-pcd (i.\-e. funnel\-\_\-20\-\_\-30.\-pcd). Not using this naming convention may result in corrupted or unusable database.
\item P\-C\-D files must represent previously segmented objects and must be expressed in a local reference system, i.\-e. a system centered at the object base. This system must be consistent with all the P\-C\-Ds provided.
\item P\-C\-Ds should have stored the viewpoint location (coordinates of where the sensor was positioned during acquisition) inside their sensor\-\_\-origin\-\_\- member for optimal results, although this is not mandatory
